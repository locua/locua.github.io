* Project Specification
** Project overview
In this project I will research the technical and ethical factors of microtargeting with psychographics and more widely the intersection of online experience and politics. I will create a series of web experiments that will demonstrate and posit some of the ethical and technical dilemmas of conducting political campaigning with the digital tools and techniques available today. Alongside this I will conduct extensive data retrieval and analysis from online sources to complement the former. I would like to find a specific experimental psychological / political science study to follow, conduct my own experiments and relevant data mining and analysis. The topic of political advertising is a hard data set to interact with so I think I will change this by the end of reading week.

I would like to have two rounds of data gathering from these web based experiments. One in a controlled lab environment and one where participants are doing the experiments in their own time online (in the wild). I will then compare these two to see how these two environments affect the data given as this may offer insights regarding the way we naturally consume digital media.

I would like to produce a web based outcome which will present an overall summary of the project and a formalised set of theoretical and contextual findings based on the results of my technical implementations and data analysis. This will take the form of an interactive and reactive essay published online that will present my analysis. 

** Technologies and Materials
I will do a series of online experiments to collect data and then analyse and present the findings. These will take the form of in browser psychology experiments where psychological profiles are constructed and relevant political statements are presented which adapt to the persons profile. I will use my web development skills to design the applications, this will include server side and front end programming as well as database management to collect the data. I would also use visualisation techniques and some statistical analysis if enough data is collected to allow for a holistic analysis of the overall data set. I will design the experiments ethically, making sure to get full consent from participants and ensuring the necessary anonymisation and eventual deletion of any raw data I collect. Analysis of meta and “soft” data could also open additional doorways of inquiry which could be explored or just illuminated as potential adjacent areas of research. Various web frameworks will be looked at such as Flask (python) with a MongoDB or SQL backend as well as looking at jsPsych (specifically for psychology experiments). 

A second pathway of technical exploration will be through scraping data from social media to glean insights and to provide an additional dataset and subsequent analysis to compare to my own data collection. To do this I will use python and its various libraries for data processing and analysis, I will use iPython / jupyter notebooks to organise and carry out this work.
** Context and research

I have been reading around the psychology of influence, specifically through the lense of psychologically and demographically targeted advertising. This is where big data has been used to specifically target individuals with political adverts tailored to their psychological and demographic personal profile. I would like to explore the ethics of this kind of targeting from a technical perspective and through practical demonstration of the kinds of techniques that have been utilised as well as the possibility they could be utilised for social good. 

These themes may be changed after reading week, if I find a specific study in experimental psychology and political science that I can continue as part of this project I will adapt the theoretical research questions

Particularly my experience with python in a web context and also with digital signal processing last year (Perception and Multimedia Computing, audio track). I will consolidate my python knowledge generally as well as my experience with the Flask framework, MongoDB and mysql. I will also use python for its great support for data processing capabilities; for visualisation, statistical analysis and text processing / extraction.

* Project Specification mark 2:
** Project overview

In this project I will create a browser based system for spatial tabletop computer interaction. It will be web based interactive {computing} environment where one can /write programs/ by arranging coloured objects on the table. When put in particular patterns this will trigger an affect. This could be displaying an image or connecting objects in the space, displaying a text editor or a web page. The fundamental idea here would be an interaction model based of physical placement and movement of objects on a 2d surface space which then reacts live. 

#+CAPTION: sketch 1
[[file:setup.jpg]]

A set of 4 or 5 different symbols each represented by coloured dots will be created and this will control the environment. These will be printed on paper or be individual pieces which are place around the surface. The relative positioning and interaction of these dots will cause various effects. 

I thought of the possibility of implementing a Turing Machine but the first thing would be creating simple interactive models; this could also be a drawing or modelling tool or game. I would like to create a base setup which would allow for easy expansion and be modular in it's essence, first creating a base Computer vision and display system with an API following this an interaction design style game experiment. This could be something relatively simple but it would be a game that facilitates an exploration of the cognitive aspects of /human computer interaction/.

** Technologies and Materials

The setup will consist of a nodejs server serving client side javascript. This will include an [[https://docs.opencv.org/3.4/d4/da1/tutorial_js_setup.html][emscripten]] or [[https://webassembly.org/][WebAssembly]] compiled version of the opencv library. For the projection mapped surface I will do this with Javascript, HTML and the DOM and potentially with [[https://reactjs.org/][React Js]]. Scripting of the language model and base system API will be done in Javascript as well as the games implemented. I will look into React for the interface but may end up using the standard Javascript, HTML and DOM elements.

The motivation for having a browser based system is accessibility and portability as a compromise over speed. As PaperPrograms demonstrates a responsive enough system is doable through the browser and with this it allows for easy usage by anyone with a projector, webcam and computer. In keeping with [[https://paperprograms.org/][PaperProgram's]] system I will use postgresql for a database system if this ends up being a necessary part of the overall piece. I should also formally mention PaperPrograms, which uses this nodejs, opencv and postgresql stack, as my main technical reference for the project.

Hardware will consist of my laptop running locally the node server and client system as well as a projector; PaperPrograms recommend 1080p and at least 2000 Limns. Colour printing and potentially colour pens or some other source of coloured paper. Colour printing should be good for consistent colour.

** Context and research

My main point of reference is [[https://dynamicland.org][Dynamicland]] cite:VictorKayDynamicLand, an immersive collaborative computer system based in Oakland (US), and more specifically PaperPrograms, a web implementation of one element of Dynamicland's /Realtalk OS/. /PaperPrograms/ is a camera, projector and computer setup where colour pattern coded  pieces of paper are recognised by a camera and corresponding Javascript programs are then run and the results of which are projected onto the same surface. This creates a loop where program-object has physical representation on a surface; it's position and relation to other programs can have effects on the overall system.

Through the project I would like to explore the cognitive aspects of /Human Computer Interaction/ cite:SharpHelen2019IDBH, particularly behavioural influence. This could be done by creating a simple interactive game/s or experience which illustrates some of the psychological and cognitive phenomena of HCI.
